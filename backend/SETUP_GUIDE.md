# FieldVision AI Backend Setup Guide (Windows PowerShell)

## 📋 Project Overview

This backend uses **FastAPI** (a modern Python web framework) and **Uvicorn** (a server) to create a REST API for FieldVision AI.

---

## 🔧 STEP 1: Open PowerShell

1. Press `Windows Key + X`
2. Click "Windows PowerShell" or "Terminal"
3. Navigate to your backend folder:
   ```powershell
   cd "C:\Users\kjosh\OneDrive\Desktop\FieldVision AI\backend"
   ```

---

## 🐍 STEP 2: Create Virtual Environment

A virtual environment keeps your project dependencies separate from your system.

```powershell
python -m venv venv
```

**What this does:**
- Creates a folder called `venv` in your backend directory
- This folder contains a Python installation isolated for this project

---

## ✅ STEP 3: Activate Virtual Environment

**Important:** You must activate the venv every time you open PowerShell!

```powershell
.\venv\Scripts\Activate.ps1
```

**You should see `(venv)` appear at the start of your PowerShell prompt:**
```
(venv) PS C:\Users\kjosh\OneDrive\Desktop\FieldVision AI\backend>
```

**If you get an error about execution policy:**
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
Then run the activate command again.

---

## 📦 STEP 4: Install Dependencies

With venv activated, install required packages:

```powershell
pip install -r requirements.txt
```

**This installs:**
- ✅ FastAPI - Web framework
- ✅ Uvicorn - ASGI server
- ✅ python-dotenv - Environment variable management

**Verify installation:**
```powershell
pip list
```
You should see fastapi and uvicorn in the list.

---

## 🚀 STEP 5: Run the Server

**Make sure venv is still activated** (you should see `(venv)` in your prompt)

```powershell
python -m uvicorn main:app --reload
```

**Expected output:**
```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Application startup complete
```

---

## 🌐 STEP 6: Test the Server

1. Open your browser and go to: `http://127.0.0.1:8000`
2. You should see: `{"message": "FieldVision AI Backend Running"}`

### API Endpoints Available:

- **GET** `http://127.0.0.1:8000/` - Welcome message
- **GET** `http://127.0.0.1:8000/health` - Health check
- **POST** `http://127.0.0.1:8000/analyze?text=hello` - Analyze text

### View Interactive API Documentation:

Go to: `http://127.0.0.1:8000/docs`

This is Swagger UI - you can test API endpoints directly in your browser!

## ⚠️ Common Beginner Mistakes (Avoid These!)

### ❌ Mistake 1: Files Inside `.venv` Folder
**WRONG:**
```
backend/
├── .venv/
│   ├── main.py          ← DO NOT PUT HERE!
│   └── ai.py            ← DO NOT PUT HERE!
```

**CORRECT:**
```
backend/
├── venv/
├── main.py              ← PUT HERE!
├── ai.py                ← PUT HERE!
├── .env
└── requirements.txt
```

### ❌ Mistake 2: Wrong File Extensions
**WRONG:**
- `main.py.txt` - This is a TEXT file, not Python!
- `main.py.bak` - Wrong extension
- `MAIN.PY` - Python is case-sensitive on some systems

**CORRECT:**
- `main.py` - Exactly this name

**How to check in VS Code:**
- Look at the file tab - it should show just "main.py"
- Right-click file → "Change Language Mode" → select "Python"

### ❌ Mistake 3: Forgetting to Activate venv
**WRONG:**
```powershell
pip install fastapi  # ✗ Installs to system Python, not venv
```

**CORRECT:**
```powershell
.\venv\Scripts\Activate.ps1  # Activate first
pip install fastapi          # Now installs to venv
```

**Check if venv is active:**
- You should see `(venv)` at the start of your PowerShell prompt

### ❌ Mistake 4: Uvicorn Not Installed
**Error you might see:**
```
ModuleNotFoundError: No module named 'uvicorn'
```

**Solution:**
```powershell
pip install uvicorn[standard]  # [standard] includes extra features
```

### ❌ Mistake 5: Port Already in Use
**Error:**
```
Address already in use
```

**Solution - Use a different port:**
```powershell
python -m uvicorn main:app --reload --port 8001
```

Then visit: `http://127.0.0.1:8001`

## 📝 File Descriptions

### `main.py` - Main Server File
- FastAPI application entry point
- Defines all API endpoints
- Runs the Uvicorn server

### `ai.py` - AI Module
- Placeholder for AI logic
- Will integrate Gemini API here
- Keep this separate for clean code organization

### `.env` - Environment Variables
- Stores API keys (keep secret!)
- Configuration settings
- Database URLs

### `requirements.txt` - Dependencies
- Lists all Python packages needed
- Used with `pip install -r requirements.txt`

### `venv/` - Virtual Environment Folder
- **DO NOT COMMIT TO GIT**
- **DO NOT EDIT MANUALLY**
- Auto-generated by Python
- Add to `.gitignore`

## 🐛 Troubleshooting

### PowerShell Script Execution Error
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Module Not Found Errors
```powershell
# Make sure venv is activated, then:
pip install --upgrade pip
pip install -r requirements.txt
```

### Can't connect to http://127.0.0.1:8000
1. Check server is running (look for "Uvicorn running on...")
2. Copy the exact URL from the terminal
3. Make sure server isn't on a different port (e.g., :8001)

### ModuleNotFoundError: No module named 'fastapi'
```powershell
# Verify venv is activated (should see (venv) in prompt)
pip install fastapi uvicorn
```
